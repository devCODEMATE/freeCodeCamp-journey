# 📝 Markdown to HTML Converter

> JavaScript Certification Project — freeCodeCamp

## 🇪🇸 Español

Este es el proyecto número 89 de mi recorrido de JavaScript en freeCodeCamp y forma parte de los proyectos requeridos para obtener la certificación.

El objetivo fue construir una aplicación que convierta texto escrito en Markdown a HTML. La aplicación procesa diferentes patrones de formato y muestra tanto el código HTML generado como una vista previa del resultado.

### Funcionalidades

- Conversión de encabezados Markdown a etiquetas HTML.
- Conversión de texto en negrita.
- Conversión de texto en cursiva.
- Conversión de imágenes.
- Conversión de enlaces.
- Conversión de citas en bloque.
- Visualización del código HTML generado.
- Vista previa dinámica del contenido convertido.

- `.replace()`

### Lo que aprendí

- Crear expresiones regulares para reconocer patrones de texto.

## 🇺🇸 English

This is project number 89 in my freeCodeCamp JavaScript journey and one of the required projects for earning the certification.

The goal was to build an application that converts Markdown text into HTML. The app processes different formatting patterns and displays both the generated HTML code and a preview of the result.

### Features

- Convert Markdown headings into HTML elements.
- Convert bold text.
- Convert italic text.
- Convert images.
- Convert links.
- Convert blockquotes.
- Display the generated HTML code.
- Dynamically preview the converted content.

### Concepts practiced

- HTML
- CSS
- JavaScript
- DOM Manipulation
- Event Listeners
- Regular Expressions
- Capturing Groups
- `.replace()`
- Template literals
- `innerHTML`
- Text processing
- Dynamic rendering
- Markdown-to-HTML transformation

### What I learned

- Interpret basic Markdown syntax rules.
- Create regular expressions to recognize text patterns.
- Use capturing groups to preserve parts of the original content.
- Replace Markdown syntax with HTML elements.
- Update an interface while the user types.
- Separately display generated code and its visual output.
- Divide application logic into smaller transformation steps.
- Complete an evaluated project by following requirements and automated tests.
